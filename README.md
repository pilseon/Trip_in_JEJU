<img src="https://github.com/user-attachments/assets/1c871a74-31f9-498a-9bdf-0fda3adf97d4" style="border-radius: 10%;">

* 배포 Server URL : https://kkyul.site

## 🖥️ 프로젝트 설명

* 제주도를 여행하는 여행객을 위한 추천 장소 제공
* 다양한 여행객들의 리뷰를 통해 여행동선 최소화
* 여행 일정을 기록하고, 캘린더를 통한 여행 날짜 확인
* 해당 장소에 직접 방문한 사람들의 생생한 리뷰 확인

#### ⚡️기능 구현
#### AI CHAT 봇
	- CHAT 봇 구현 
        - 일반 GPT와 같은 생성형 AI로 사용자의 질문에 대한 답을 전달
	- 로그인, 비로그인 유저도 사용 가능

#### 이벤트 CRUD
	- 관리자 권한으로 이벤트 생성 구현
	- 생성한 이벤트는 사용자가 확인
	- 이벤트 수정 및 삭제는 관리자만 사용 가능하도록 구현

#### 미니게임
	- 로그인 했을 때만 이용 가능
	- 간단한 조작을 이용한 게임 구현
	- 점수에 따른 랭킹 등록

#### GPS (MAP)
	- Kakao API를 활용한 지도 구현
	- Kakao MAP에 등록된 지역등을 활용한 검색 기능 구현
	- 검색 창에 타이핑시 해당 자음 및 모음에 맞는 Kakao MAP에 등록된 지역 연관검색어 출력
	- 현재 위치확인 기능 구현
	- 사용자의 위치와 카테고리에 등록된 위치에 따른 리뷰 작성 권한 부여

#### 메인페이지
	- 생성한 이벤트 수에 따라 메인페이지의 이벤트 배너가 출력

#### 리뷰작성
	- GPS를 활용한 리뷰 작성 기능 구현

#### 게시물 검색
	- 카테고리에 등록된 게시물 검색 기능 구현
	- 타이핑 시 해당 자음 및 모음에 맞는 연관검색어 출력 (카테고리에 저장된 것만)
	- 검색한 게시물 클릭시 해당 DETAIL FORM으로 이동 구현
	- 검색 결과가 없을 경우 검색 결과 없음 디자인 구성

#### 마이페이지
	- 전체 일정에 등록된 일정이 없을 경우 해당 날짜에 등록된 축제 없음 디자인 구성
	- 찜 목록에 등록된 일정이 없을경우 찜 목록 없음 디자인 구성

#### 유저페이지
	- GPS로부터 등록된 방문한 카테고리들은 출력
	- 방문 기록 및 리뷰 기록을 클릭하여 FORM 이동 구현
	- 리뷰 등록 버튼을 클릭하여 해당 DETAIL FORM으로 이동 후 리뷰 작성부분으로 앵커 기능 구현
	- 리뷰 등록이 완료되었을 경우 리뷰 등록 대신 리뷰등록 완료 문구 출력
	- 리뷰 등록이 완료된 게시글에는 리뷰 기록에 저장
	- 리뷰 기록에 있는 게시글 클릭 시 해당  DETAIL FORM으로 이동 구현

#### 게시물 LIST
	- 카테고리 별로 이동할 수 있도록 버튼 구현
	- MAP을 활용하여 게시글 HOVER시 해당 장소로 MAP 이동과 함께 소형의 카드 출력
	- 카드의 자세히보기를 클릭하여 해당 DETAIL FORM으로 이동

#### 게시물 DETAIL
	- MAP에 해당 위치를 표시
	- 방문한 적이 있는 곳에 리뷰 작성하도록 구현
	- 리뷰 작성 시 이중 작성 불가하도록 구현

#### 게시물 작성
	- 위치를 장소 대신 주소로 받을 수 있도록 구현
	- MAP에 장소를 입력하여 해당 장소와 주소를 동시에 받을 수 있도록 구현 
	- 장소 입력 시 연관검색어 출력

### 👻어근혁

#### 🚧 Swiper 슬라이더 문제 분석 및 해결 🚨

🛑 원인\
메인 페이지에 Swiper 슬라이더를 적용한 후, 처음에는 각 테마별 랜덤 추천 기능에서 슬라이더가 잘 작동 하였으나 페이지 하단에 새로운 이벤트 콘텐츠를 추가한 이후부터 슬라이더가 정상적으로 동작하지 않는 문제가 생김
- Swiper 기능의 비활성화:\
새로운 콘텐츠가 추가된 이후, Swiper 슬라이더가 더 이상 동작하지 않게 됌.


- 이미지 크기 불일치:\
첫 번째 슬라이드 이미지가 비정상적으로 커지면서 레이아웃이 깨지고, 사용자 경험이 저하되는 문제가 발생했음.
  예상 되었던 문제 원인으로는  Swiper 슬라이더가 새롭게 추가된 콘텐츠와 함께 정상적으로 초기화되지 않거나,
  특정 DOM 구조와 CSS가 충돌하면서 발생한 것으로 보였고 특히, 새로운 콘텐츠가 추가되면서 Swiper 슬라이더가 올바르게 재렌더링되지 않았기 때문에 첫 번째 슬라이드 이미지가 비정상적으로 확대되는 문제로 이어졌을 가능성이 크다고 생각하였음.

🚥 해결\
이 문제를 해결하기 위해 Swiper 라이브러리를 그대로 사용하는 대신,
- 커스텀 슬라이드 제작:\
Swiper의 기본 기능을 사용하는 대신, DOM 구조와 스타일을 직접 제어할 수 있는 커스텀 슬라이드를 새로 제작하였고, 이로 인해 Swiper의 초기화 문제를 피하도록 하였음


- 동적 콘텐츠 업데이트 처리:\
페이지에 새로운 콘텐츠가 추가될 때마다 슬라이더가 올바르게 재렌더링되고 초기화되도록 코드를 수정하혔고, 이를 통해 슬라이더가 정상적으로 동작하도록 수정 하였음


- CSS 및 자바스크립트 최적화:\
슬라이더와 관련된 CSS와 자바스크립트 코드를 최적화하여 다른 요소들과의 충돌을 최소화하였음
  이렇게 해서 Swiper 슬라이더가 다시 정상적으로 동작하게 되었고, 페이지 전체가 안정적으로 작동하도록 개선하였음.
